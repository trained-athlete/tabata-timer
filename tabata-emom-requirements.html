TABATA & EMOM Timer — Requirements

Single-file app: tabata-emom-timer.html · Last updated: 21 Apr 2026

✅ Implemented ⚠️ Partial ❌ Not built
1 · Workout modes
TABATA Tabata timer
  • ✅ Configurable work interval (default 20 s)
  • ✅ Configurable rest interval (default 10 s); can be set to 0
  • ✅ Configurable number of rounds (default 8)
  • ✅ Clearly distinguish work vs. rest phase visually (colour + glow)
  • ✅ "Last round" alert before final set
  • ✅ Warning beeps 3 seconds before each phase transition
EMOM EMOM timer
  • ✅ Configurable interval duration (default 60 s)
  • ✅ Configurable total number of rounds (default 10)
  • ✅ Interval can be set to values other than 60 s (free-form min:sec)
  • ✅ Audio/visual cue at the start of each new interval
  • ✅ "Last round" alert before final interval
  • ✅ Round counter shown prominently during workout
  • ✅ Total elapsed time displayed alongside interval countdown
2 · Timer controls (shared)
Controls
  • ✅ Start / Pause / Resume
  • ✅ Stop & reset to initial state
  • ✅ Skip current interval
  • ✅ 3-second countdown before timer starts
Display
  • ✅ Large, high-contrast time digits (96 px / 72 px mobile)
  • ✅ Current round / total rounds indicator
  • ✅ Current phase label (Work / Rest / Round N)
  • ✅ Stats row: elapsed time, remaining time, round counter
  • ✅ SVG progress ring per interval
  • ✅ Phase-specific background glow (radial gradient)
Audio
  • ✅ Beep on interval start/end (distinct frequencies)
  • ✅ Countdown beeps 3 s before phase transitions
  • ✅ Last-round warning sound (triple-beep pattern)
  • ✅ "Done" fanfare — 4-note ascending melody
  • ✅ Toggle audio on / off (per mode)
3 · Configuration UI
  • ✅ Mode selector: Tabata / EMOM (tab buttons in header)
  • ✅ Numeric inputs for time values (min : sec cells)
  • ✅ +/− stepper buttons for round count
  • ✅ Input validation — prevent zero total duration
  • ✅ Workout preview: total duration, work time, rest time (Tabata) or per-round time (EMOM)
  • ✅ Settings persist across page reloads (localStorage)
  • ⚠️ Preset configurations — history-based "Load" only, no named presets
  • ✅ Toast notifications for user feedback
4 · UI & layout
Responsiveness
  • ✅ Mobile-first responsive layout (720 px breakpoint)
  • ⚠️ Landscape mode — responsive but no dedicated layout
  • ✅ Readable from 3+ metres (gym wall scenario)
Visuals
  • ✅ Dark theme with custom colour palette (lime work, red rest, blue EMOM)
  • ✅ Color change between work and rest phases (text, ring, glow)
  • ✅ SVG progress ring per interval with smooth transitions
  • ✅ CSS animations: pulse-ring, flash-in, countdown-pop
  • ✅ Phase banner clearly visible at top of timer panel
  • ✅ Idle state prompt ("Configure and press start")
5 · Technical requirements
  • ✅ Pure JavaScript — single HTML file, no server required
  • ✅ Accurate timing via performance.now() accumulator (50 ms tick interval)
  • ✅ Web Audio API for beep generation (oscillator-based, no audio files)
  • ✅ Wake Lock API to prevent screen sleep during workout
  • ✅ Wake lock re-acquired on tab visibility change (iOS support)
  • ⚠️ Runs with screen locked / app in background (timer continues, no service worker)
  • ✅ localStorage for persisting settings and workout history
  • ⚠️ External dependency: Google Fonts (Bebas Neue, DM Mono, DM Sans)
  • ✅ Works on Chrome, Firefox, Safari, Edge (standard APIs)
  • ✅ Keyboard shortcuts: Space start/pause · R reset · S skip
6 · Workout log & history
  • ✅ Auto-save completed workouts with date, config & duration
  • ✅ Modal list view of past workouts (max 50 entries, auto-trimmed)
  • ✅ Load / replay a past workout from history
  • ✅ Display mode, total duration, rounds & config per entry
  • ✅ Modal closes on overlay click
