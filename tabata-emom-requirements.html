JavaScript web app — requirements

TABATA & EMOM timer · Reference: timer.crosshero.com

1 · Workout modes
TABATA Tabata timer
  • Configurable work interval (default 20 s)
  • Configurable rest interval (default 10 s); can be set to 0
  • Configurable number of rounds (default 8)
  • Multiple independent Tabata sets in sequence
  • Clearly distinguish work vs. rest phase visually
  • "Last round" alert before final set
  • Mid-workout audio signal option
EMOM EMOM timer
  • Configurable interval duration (default 60 s)
  • Configurable total number of rounds
  • Interval can be set to values other than 60 s (e.g. every 2 min)
  • Audio/visual cue at the start of each new minute/interval
  • "Last round" alert before final interval
  • Round counter shown prominently during workout
  • Total elapsed time displayed alongside interval countdown
2 · Timer controls (shared)
Controls
  • Start / Pause / Resume
  • Stop & reset to initial state
  • Skip current interval
  • Countdown before timer starts (e.g. 3-second lead-in)
Display
  • Large, high-contrast time digits
  • Current round / total rounds indicator
  • Current phase label (Work / Rest)
  • Total workout time remaining or elapsed
Audio
  • Beep on interval start/end
  • Voice audio option ("3, 2, 1, Go!")
  • Distinct sounds for work vs. rest transitions
  • Toggle audio on / off
  • Last-round warning sound
3 · Configuration UI
  • Mode selector: Tabata / EMOM (+ AMRAP, For Time as future tabs)
  • Numeric inputs or scroll-picker for time values (min : sec)
  • Input validation — prevent zero total duration
  • Preset / saved workout configurations
  • Ability to add multiple sets of the same type in sequence
  • Reset all settings to defaults
  • Settings persist across page reloads (localStorage)
  • Summary preview: total workout duration shown before start
4 · UI & layout
Responsiveness
  • Mobile-first responsive layout
  • Landscape mode — large timer fills screen width
  • Readable from 3+ metres (gym wall scenario)
  • Dark / light mode toggle
  • Full-screen mode support
Visuals
  • Color change between work and rest phases
  • Progress bar or ring per interval
  • Smooth countdown animation
  • Phase label clearly visible at a glance
  • Motivational label or phase name customisation
5 · Technical requirements
  • Pure JavaScript — no server required; fully client-side
  • Accurate timing via setInterval + drift correction, or AudioContext clock
  • Web Audio API for beep generation (no external audio files needed)
  • Runs with screen locked / app in background (where browser allows)
  • Wake Lock API to prevent screen sleep during workout
  • localStorage for persisting settings and workout history
  • PWA-ready: manifest + service worker for offline use
  • No external dependencies required for core timer logic
  • Works on Chrome, Firefox, Safari, Edge (latest 2 versions)
  • Keyboard shortcuts: Space = start/pause, R = reset, S = skip
6 · Workout log & history
  • Auto-save completed workouts with date & configuration
  • List view of past workouts
  • Replay / re-run a past workout from history
  • Clear individual entries or full history
  • Display total rounds completed and duration
